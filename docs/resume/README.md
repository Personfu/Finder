# Resume System — Usage Guide

Quick reference for using the resume composer day-to-day. For the design
rationale, read [strategy.md](strategy.md). For the facts of record, read
[canon.md](canon.md).

## What's here

Three layers feed one composer that produces tailored resumes per role archetype.

```
data/canon.json                    facts of record (PII; gitignored)
data/skill_library/*.md            bullets, one file per role (committed)
config/archetypes.json             archetype definitions: intros, domain order, skills (committed)
scripts/_resume_style.py           constants: tier order, banned verb/phrase/punctuation lists

scripts/resume_compose.py          slices canon + library → markdown
scripts/md_to_docx.py              markdown → styled DOCX (needs python-docx)
scripts/generate_generic_resumes.py   chains both for every archetype

data/generic_resumes/              md + docx per archetype, off-the-shelf (gitignored)
```

## The 7 shipped archetypes

Archetypes are defined in `config/archetypes.json`, not in code. The shipped
example ships seven, built around a deliberately broad fictional persona
(James Bond) to show one career tailored into very different jobs:

| Archetype | Skills-block leads with | Use for |
|---|---|---|
| `close-protection` | Close Protection & Physical Security | Close/executive protection (**default if unsure**) |
| `corporate-security-director` | Leadership & Team Command | Director of Corporate Security / CSO |
| `field-investigations` | Intelligence & Investigations | Investigator / field operative |
| `diplomatic-liaison` | Languages & Diplomacy | Diplomatic security / liaison |
| `intelligence-analyst` | Intelligence & Investigations | Intelligence analyst |
| `protective-driver` | Advanced & Tactical Driving | Executive protective / precision driver |
| `luxury-hospitality` | Hospitality & Connoisseurship | Luxury brand ambassador / sommelier / hospitality |

Replace these with archetypes that fit your own career; see
[strategy.md](strategy.md) and the `_doc` note at the top of
`config/archetypes.json` for how. Each archetype emits an identity-first intro
at the top and reorders the Skills section so the most relevant domain leads
(matters for ATS keyword scanning; see [FIND.md](FIND.md)).

**To add an archetype** (no code edit needed): add a block under `archetypes`
in `config/archetypes.json` with an `intro` and a `domain_order`, add any new
`domains` entries it needs, tag your skill-library bullets with
`archetypes=<name>`, then run with `--role-archetype <name>`.

## Three workflow paths

### Path 1 — Grab a generic (60 seconds)

When you need a resume now and don't have a specific JD to tailor against.

```powershell
# Refresh generics if canon or library changed since last regen
python scripts/generate_generic_resumes.py

# Pick an archetype, open the DOCX, tweak in Word if needed, submit
explorer data\generic_resumes\close-protection.docx
```

Want to see what these look like before running anything? The committed set
(built from the example persona) lives in
[`data/generic_resumes/`](../../data/generic_resumes/) — one `.md` + `.docx`
per archetype, regenerated by `scripts/generate_generic_resumes.py`.

### Path 2 — JD-tailored (10–15 min)

When you have a specific JD URL and want it dialed in for that role.

```powershell
# 1. Pick the archetype based on the role you're targeting (see table above)
python scripts/apply_pipeline.py --jd-url "<URL>" --role-archetype <archetype>

#    apply_pipeline.py:
#    - fetches the JD (Greenhouse/Lever JSON API; HTML fallback)
#    - invokes resume_compose.py → packet_dir/master_resume.md
#    - runs ats_overlap.py → keyword report
#    - scaffolds applications/<slug>/

# 2. /finder:apply (the skill) drafts tailored_resume.md + cover_letter.md
#    against the JD with CAR + banned-list rules enforced

# 3. /finder:publish <slug> — pushes to Google Docs, appends tracker row
```

### Path 3 — Update canon or a bullet

When you have new accomplishments or want to fix a date.

```powershell
# Edit the source
notepad data\canon.json                       # facts of record (PII)
notepad data\skill_library\mi6_field.md       # bullets for one role

# Re-run tests
python tests\test_resume_compose.py

# Refresh all 7 generics
python scripts\generate_generic_resumes.py

# (Optional) Re-mirror to vault — manual via Obsidian MCP today
```

## Editing the bullet library

Each `data/skill_library/<role_id>.md` (with `role_id` matching an
`employment[].id` in canon) has TOML frontmatter + tagged bullets:

```markdown
+++
role_id = "mi6_field"
+++

[domain=crisis-negotiation, tier=proven, archetypes=diplomatic-liaison,corporate-security-director,intelligence-analyst]
Negotiated the release of two detained officers across hostile borders inside
36 hours, improvising transit and cover with no support on the ground.
```

**Tag fields:**

- **`domain`**: a competency domain defined in `config/archetypes.json` (the Bond example uses `protective-ops, surveillance, firearms-tactics, tactical-driving, crisis-negotiation, intelligence-analysis, languages-diplomacy, leadership, maritime-ops, gastronomy-luxury`). Determines which Skills section line the bullet contributes. Define your own.
- **`tier`** — `proven | adjacent | aspirational`. Matches the fit-model vocabulary in `config/job_search_targets.json`. Proven bullets are never trimmed when budget exceeds.
- **`archetypes`**: comma-list. Which target archetypes this bullet surfaces for. A bullet tagged `close-protection` shows up in the close-protection generic, etc.

Field order in the tag is now order-independent; `archetypes` must be last (its value contains commas).

## Quality guardrails (enforced by the pipeline)

All defined in `scripts/_resume_style.py` and enforced at two layers:

| Guardrail | Where enforced |
|---|---|
| `BANNED_VERBS` (leveraged, utilized, spearheaded, etc.) | Apply prompt Step 5 grep; library authoring discipline |
| `BANNED_PHRASES` (robust, comprehensive, results-driven, etc.) | Apply prompt Step 5 grep |
| `BANNED_PUNCTUATION` (em-dash) | Composer auto-scans its output; warns to stderr if any em-dash leaks |
| CAR framework | Apply prompt Step 3 Voice section |
| Sentence-length variance | Apply prompt Step 5; defeats stylometric AI detection |
| Tier sort (proven > adjacent > aspirational) | Composer; refuses to drop proven bullets when trimming to budget |
| Per-archetype Skills block ordering | Composer; matches each archetype's `domain_order` in `config/archetypes.json` |

## Format spec

The composer's markdown matches what `md_to_docx.py` expects, which produces:

| Element | Style |
|---|---|
| Name | 16pt Bold, ALL CAPS, centered, horizontal rule under it |
| Contact | 10pt centered |
| Intro | identity-first, archetype-specific |
| Section headers | 12pt Bold Dark Blue (#1F4E78), ALL CAPS |
| Company | 11pt Bold, location right-tab-aligned |
| Title | 10.5pt, dates right-tab-aligned |
| Bullets | `•` glyph + tab + content, hanging indent |
| Skills | Bold `Category:` + plain content |
| Margins | 0.5" all sides |
| Body | 10.5pt default |
| Target length | ≤2 pages (current generics: 860–1010 words) |
